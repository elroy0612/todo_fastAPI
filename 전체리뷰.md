## 전반총평

1. `.env`는 .gitignore에 등록해야함. 절대로 깃허브 같은 공개위치에 올라가면 안 됨. 회사에서 이런 실수하면 바로 짤림.

2. `async-yeild 패턴`에 대해 공부해야해

3. db를 코드의 모델로 다루는 방식에 익숙해져야해. 데이터와 테이블을 각각의 추상적 모델로 다루는 것. + ORM의 개념에 대해 익숙해져야해.

4. 결국 정보를 처리하는 것이 전부임. 데이터가 있고, 그 데이터를 어떻게 사용자가 사용할수 있도록 Delivery하느냐가 핵심임. 그러기 위해 CRUD를 만드는 것이고, 지금 단계에서는 그게 전부임.

5. `레퍼지토리 패턴`에 대해 익숙해지셈. 이렇게 DB내 실제 데이터로 접근하는 계층 구조나 방법론에 따라 여러가지 디자인패턴들이 있는데, 지금 단계에서는 레포지토리 패턴만 주로 사용.

6. `Pydantic` 은 왜 쓰는 것인지 찾아봐.

7. `의존성`에 대한 개념이 있어야함. 그러기 위해서 의존성 주입 등을 왜 하는지 등에 대해서도 생각해봐

8. `트랜잭션`이라는 개념에 대해서 공부해야해

## 코드리뷰.

1. 지금 service.py 에서는 Integrity에러만 잡고 있고, 전부 다 `500` 에러코드만 반환하고 있음. 이러면 프론트 입장에서는 전부 다 'Internal Server Error' 로 수신 받아서, 어떤 에러인지 알기가 굉장히 어려워. 경우에 따라 401, 403, 404, 422, 503 등 다양한 에러코드로 반환해줘야해.

2. 지금 Repository 에 CRUD가 불완전함. add, get_by_id, list_desc 는 있는데, 수정과 삭제 메서드가 없음.

3. 현재 에러 발생 시 추적이 안 되고 있음. 너를 위해서라도 에러 로그를 탄탄히 잘 싸아봐. `logging` 라이브러리에 대해 공부해보셈.

4. 현재 Router 엔드포인트에 생성 라우터만 있음. 나머지도 CRUD에 맞게 적절히 추가되어야헤.

5. 지금 수준에서는 필요없을 수 있지만, `pytest` 등 테스트 라이브러리들을 도입해보고 써볼 생각을 해봐.

6. CORS 설정은 .env 환경 변수로 관리하는 것이 더 유리할 수 있음. 반박 환영
